# GakrCLI Settings Reference

This folder contains a fully-annotated `settings.json` template and this companion
reference, generated by reading the live schema in `src/utils/settings/types.ts`,
the feature flags in `scripts/build.ts`, and your existing `.gakrcli/settings.local.json`.
If any field in this file gets out of sync with `SettingsSchema`, update both.

---

## How Settings Are Loaded

| Layer | Path | Override? |
|-------|------|-----------|
| Policy / MDM | Managed-settings (enterprise) | Yes — highest precedence |
| Repo `.gakrcli/settings.json` | `.gakrcli/settings.json` | Yes — layer 1 |
| Workspace `settings.json` | `~/.gakrcli/settings.json` | Yes — layer 2 |
| Built-in defaults | Schema.default() | No |

Higher-precedence layers win on conflict. `pluginConfigs`, `permissions`, `env`, and
`sandbox` are *sourced* from all layers and merged.

## What You Can Toggle Right Now (open build, June 10 2026)

### Always-on behavior (no flag needed)
These are controlled **only** by `settings.json` — the build flag is irrelevant.

- **TEAMMEM** — enabled at build (`scripts/build.ts`), but you can disable it from settings
- **TRANSCRIPT_CLASSIFIER** — auto/classifier permission mode is on by default; disable with `autoMode.disable`
- **CACHED_MICROCOMPACT** — invisible optimization; nothing to toggle in settings
- **HOOK_PROMPTS** — on via build; hooks that call `requestPrompt` work automatically

### Build-flagged features (require rebuild to change)
These require changing `scripts/build.ts` and re-building. The schema key exists in types.ts
but the code path is removed when the flag is `false`.

| Flag | In settings.json? | Notes |
|------|-------------------|-------|
| `KAIROS` | `"assistant": true` key | CLOSED: set flag to `true` first |
| `VOICE_MODE` | `"voiceEnabled"` | CLOSED |
| `PROACTIVE` | `"minSleepDurationMs"` / `"maxSleepDurationMs"` | CLOSED |
| `BRIDGE_MODE` | N/A | CLOSED |
| `DAEMON` | N/A | CLOSED |
| `MCP_SKILLS` | N/A | DANGER — enabling without `src/skills/mcpSkills.ts` crashes |

### Open build, compile-time enabled
These compile in by default and require a rebuild to disable.

`COORDINATOR_MODE`, `BUDDY`, `MULTI_TURN_CONTEXT`, `MONITOR_TOOL`, `EXTRACT_MEMORIES`,
`FORK_SUBAGENT`, `VERIFICATION_AGENT`, `QUICK_SEARCH`, `HISTORY_PICKER`, `TOKEN_BUDGET`,
`HISTORY_PICKER`, `AWAY_SUMMARY`, `SHOT_STATS`, `ULTRATHINK`.

## Field Groups

### 1. Permissions — `"permissions"`
Controls which Bash / WebFetch / MCP / Skill operations require confirmation
(`ask`), are auto-allowed (`allow`), or blocked (`deny`).

| Sub-key | Type | Default | Effect when `true` / set |
|---------|------|---------|--------------------------|
| `allow` | string[] | `[]` | Pattern list of allowed commands without prompting. Globs / tool prefixes supported (e.g. `"Bash(bun test:*)"`) |
| `deny` | string[] | `[]` | Pattern list of always-blocked commands. Overrides allow. |
| `ask` | string[] | `[]` | Always prompt for confirmation, even if allow-matched elsewhere. |
| `defaultMode` | enum | `"default"` | `"auto"` uses AI classifier; `"plan"` requires plan approval; `"dangerous"` bypasses most prompts (accept dialog required) |
| `allowBypassPermissions` | enum | `"none"` | `"enable"` enables dangerous-mode toggle; `"always"` suppresses toggle UI |

### 2. Context / Memory — `"context"`, `"autoMemory*"`, `"autoDreamEnabled"`
| Key | Type | Default | Notes |
|-----|------|---------|-------|
| `cleanupPeriodDays` | number | `30` | Session transcript retention. `0` = no persistence at all |
| `autoMemoryEnabled` | bool | `true` | Read/write auto-memory in project memory dir |
| `autoMemoryDirectory` | string | computed | Override default `~/.gakrcli/workspace/projects/<cwd>/memory/` |
| `autoDreamEnabled` | bool | server default | Background memory consolidation |
| `respectGitignore` | bool | `true` | File picker respects `.gitignore` |

### 3. UI / Display — `"outputStyle"`, `"prefersReducedMotion"`, `"spinner*"`, `"syntaxHighlightingDisabled"`
| Key | Type | Default | Notes |
|-----|------|---------|-------|
| `outputStyle` | string | `undefined` | Influences assistant reply format |
| `prefersReducedMotion` | bool | `false` | Disable spinner shimmer / flash effects |
| `spinnerTipsEnabled` | bool | server default | Tips under spinner |
| `spinnerVerbs` | object | see example | Customize spinner verbs |
| `syntaxHighlightingDisabled` | bool | `false` | Plain text diffs |

### 4. Providers / Models — `"agentModels"`, `"agentRouting"`, `"advisorModel"`, `"modelPicker*"`, `"fastMode"`
| Key | Type | Default | Notes |
|-----|------|---------|-------|
| `agentModels` | record | `{}` | Name → `{ base_url, api_key }` map for OpenAI-compatible providers |
| `agentRouting` | record | `{}` | Agent name → model name routing (key `"default"` is fallback) |
| `fastMode` | bool | `false` | GPT-4o-mini-class speed routing |
| `fastModePerSessionOptIn` | bool | `false` | Fast mode off by default each new session |
| `advisorModel` | string | server default | Override advisor model name |
| `effortLevel` | enum | `undefined` | `"low" | "medium" | "high" | "max"` — only models that support effort |
| `alwaysThinkingEnabled` | bool | `true` | Keep extended thinking on by default |

### 5. Auth / OAuth — `"xaa"`, `"codexOAuth"`, `"oauthClients"`, `"forceLoginMethod"`
| Key | Type | Notes |
|-----|------|-------|
| `xaa.id` | string | XAA / SEP-990 IdP client ID |
| `xaa.callbackPort` | number | OIDC loopback callback port if IdP ignores RFC 8252 |
| `codexOAuth.secretStorageNamespace` | string | Namespace for `keytar` stored codex OAuth token |
| `codexOAuth.enableLocalhostAuth` | bool | Allow localhost callback for Codex OAuth flow |
| `forceLoginMethod` | enum | `"gakrcliai"` or `"console"` |

### 6. Plugins / MCP — `"pluginConfigs"`, `"extraKnownMarketplaces"`, `"strictKnownMarketplaces"`, `"blockedMarketplaces"`
| Key | Notes |
|-----|-------|
| `pluginConfigs[<id>].mcpServers` | Per-plugin MCP server user config values |
| `extraKnownMarketplaces` | Additional marketplace sources for this repo (team-wide) |
| `strictKnownMarketplaces` | Enterprise allowlist — gate runs before download |
| `blockedMarketplaces` | Enterprise blocklist — gate runs before download |

### 7. Tool Behavior — `"sandbox"`, `"env"`, `"cleanupPeriodDays"`
`sandbox` is a separate nested schema — see `src/entrypoints/sandboxTypes.ts` for full shape.
`sandbox.enabled = true` activates sandboxed Bash execution.

### 8. Enterprise / DevOps — `"autoUpdatesChannel"`, `"minimumVersion"`, `"plansDirectory"`, `"pluginTrustMessage"`, `"channelsEnabled"`, `"allowedChannelPlugins"`, `"forceLoginOrgUUID"`, `"forceLoginMethod"`
| Key | Notes |
|-----|-------|
| `autoUpdatesChannel` | `"latest"` or `"stable"` |
| `minimumVersion` | String semver floor — prevents downgrade to stable |

### 9. Advanced / Experimental — `"skipWebFetchPreflight"`, `"otelHeadersHelper"`, `"feedbackSurveyRate"`, `"companyAnnouncements"`
| Key | Notes |
|-----|-------|
| `feedbackSurveyRate` | `0.0 – 1.0` probability per eligible session |
| `skipWebFetchPreflight` | `true` only in locked-down corporate networks |
| `otelHeadersHelper` | Path to script that prints OTel headers (dev/op tracing only) |
